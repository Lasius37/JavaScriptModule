    Récupérer les données d'un formulaire

    Pour récupérer les données d'un formulaire, on utilise l'événement submit et l'objet FormData.

    • Exemple de récupération des données : 
      <form id="monFormulaire">
    <input type="text" name="nom" />
    <input type="email" name="email" />
    <button type="submit">Envoyer</button>
</form>
      const form = document.querySelector("#monFormulaire");

form.addEventListener("submit", (e) => {
    e.preventDefault();
    const formData = new FormData(form);

    const nom = formData.get("nom");
    const email = formData.get("email");

    console.log(nom, email);
});
    • preventDefault() empêche le rechargement de la page.

    Valider une saisie utilisateur

    La validation permet de vérifier que les données saisies par l'utilisateur sont conformes aux attentes.

    • Exemple de validation d'un email : 
      function validerEmail(email) {
    const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    return regex.test(email);
}

if (!validerEmail(email)) {
    console.log("Email invalide");
}
    • On peut aussi utiliser l'API Constraint Validation du HTML5 : 
      <form id="monFormulaire" novalidate>
    <div>
        <label for="nom">Nom :</label>
        <input type="text" id="nom" name="nom" required minlength="2" maxlength="50">
        <div class="error-message" id="nom-error"></div>
    </div>

    <button type="submit">Envoyer</button>
</form>

      .error-message {
    color: red;
}

input:invalid {
    border: 1px solid red;
}

input:valid {
    border: 1px solid green;
}

      const form = document.querySelector("#monFormulaire");

form.addEventListener("submit", (e) => {
    e.preventDefault();

    // Vérification de la validité
    const nomValid = form.elements.nom.checkValidity();
    const errorElement = document.querySelector("#nom-error");

    // Message d'erreur afiché ou non
    if (nomValid) {
        errorElement.textContent = "";  // pas de message
        form.submit();  // envoi du formulaire
    } else {
        errorElement.textContent = form.elements.nom.validationMessage;
        // message natif ; on peut remplacer par une chaîne
    }
};

    Envoyer un formulaire sans recharger la page

    Pour envoyer un formulaire sans recharger la page, on utilise fetch avec les données du formulaire.

    • Exemple d'envoi de données : 
      form.addEventListener("submit", async (e) => {
    e.preventDefault();
    const formData = new FormData(form);

    const response = await fetch("/api/submit", {
        method: "POST",
        body: formData,
    });

    const result = await response.json();
    console.log(result);
});
    • fetch permet d'envoyer les données au serveur et de recevoir une réponse.

    Afficher les messages d'erreur

    Pour afficher les erreurs de validation, on peut ajouter des messages dans le DOM.

    • Exemple d'affichage d'erreur : 
      if (!validerEmail(email)) {
    const errorElement = document.createElement("p");
    errorElement.textContent = "Email invalide";
    errorElement.classList.add("error");

    form.appendChild(errorElement);
}
    • On peut ainsi utiliser des classes CSS pour styliser les messages d'erreur.

    Erreurs fréquentes

    • Oublier preventDefault() sur l'événement submit : la page se recharge.
    • Ne pas valider les données côté serveur : risque de sécurité.
    • Afficher les erreurs sans les effacer avant : les erreurs s'accumulent.
    • Oublier de gérer les erreurs de fetch : les erreurs ne sont pas visibles.
