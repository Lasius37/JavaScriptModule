    Enregistrer des données localement

    Pour stocker des données localement dans le navigateur, on utilise localStorage ou sessionStorage.

    • Enregistrer du texte dans localStorage : 
      localStorage.setItem("nom", "Alice");
    • Enregistrer un objet dans localStorage : 
      const utilisateur = { nom: "Alice", age: 25 };
localStorage.setItem("utilisateur", JSON.stringify(utilisateur));
    • localStorage persiste même après la fermeture du navigateur.
    • sessionStorage est effacé à la fermeture de l'onglet.

    Lire des données stockées

    Pour lire les données stockées, on utilise getItem.

    • Lire une chaîne de caractères : 
      const nom = localStorage.getItem("nom");
console.log(nom);
    • Lire un objet (nécessite une conversion depuis JSON) : 
      const utilisateurString = localStorage.getItem("utilisateur");
const utilisateur = JSON.parse(utilisateurString);
console.log(utilisateur.nom);
    • Si la clé n'existe pas, getItem renvoie null.

    Supprimer des données

    Pour supprimer une donnée, on utilise removeItem. Pour tout effacer, on utilise clear.

    • Supprimer une donnée : 
      localStorage.removeItem("nom");
    • Supprimer toutes les données : 
      localStorage.clear();

    Cas d'usage courant

    Voici quelques cas d'usage courants pour localStorage et sessionStorage.

    • Sauvegarder les préférences utilisateur : 
      // Sauvegarder l'identifiant
localStorage.setItem("username", "myname");

// Récupérer l'identifiant au chargement de la page
const username = localStorage.getItem("username") || "";
document.body.className = username;
    • Stocker un panier d'achat : 
      let panier = JSON.parse(localStorage.getItem("panier") || "[]");
panier.push({ id: 1, nom: "Produit 1", quantite: 2 });
localStorage.setItem("panier", JSON.stringify(panier));

    Erreurs fréquentes

    • Oublier de convertir les objets au format JSON avant de les stocker : les données ne sont pas exploitables.
    • Utiliser localStorage pour des données sensibles : risque de sécurité.
    • Ne pas vérifier si une clé existe avant de la lire : risque d'erreur si la clé n'existe pas.
    • Dépasser la limite de stockage (généralement 5 Mo) : les données ne sont pas enregistrées.

localStorage.setItem("utilisateur", utilisateur);
const utilisateurRecupere = localStorage.getItem("utilisateur");
