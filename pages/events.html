    Réagir à un événement

    Un événement est une action détectée par le navigateur : clic, touche clavier, saisie dans un champ, déplacement de la souris, envoi d’un formulaire, etc. Pour réagir à un événement, on utilise addEventListener.

    • Structure de base : 
      const btn = document.querySelector("#btn");

btn.addEventListener("click", () => {
    console.log("Clique !");
});
    • Événements associés à la souris :
      • "click"
      • "dblclick"
      • "mouseenter"
      • "mouseleave"
    • Événements associés au clavier :
      • "keydown"
      • "keyup"
    • Événements associés à un formulaire :
      • "input"
      • "change"
      • "submit"
    • Il est parfois préférable de passer par une fonction nommée (utile si on veut retirer l’écouteur plus tard par exemple) : 
      function direBonjour() {
    console.log("Bonjour");
}

btn.addEventListener("click", direBonjour);

    Comprendre l'objet Event

    Quand un événement se produit, le navigateur appelle la fonction associée et lui fournit un objet Event (souvent noté e). Cet objet contient des informations sur ce qui s’est passé. 

    const btn = document.querySelector("#btn");

btn.addEventListener("click", (e) => {
    console.log(e);
});
    • e.type : type d’événement (ex : "click")
    • e.target : élément sur lequel l’événement a eu lieu
    • e.currentTarget : élément qui écoute l’événement
    btn.addEventListener("click", (e) => {
    console.log(e.type);   // "click"
    console.log(e.target);  // l'élément cliqué
    console.log(e.currentTarget);  // l'élément qui écoute
});

    target peut être un enfant, tandis que currentTarget est toujours l’élément sur lequel l’écouteur a été placé.

    Comprendre la propagation des événements

    Quand on clique sur un élément, l’événement ne concerne pas seulement cet élément : il "remonte" dans l’arbre du DOM (du plus profond vers les parents). C’est la propagation (bubbling). 

    <div id="zone">
    <button id="btn">Bouton</button>
</div>
    const zone = document.querySelector("#zone");
const btn = document.querySelector("#btn");

zone.addEventListener("click", () => {  // déclenchée en deuxième
    console.log("zone");
});

btn.addEventListener("click", () => {  // déclenchée en premier
    console.log("bouton");
});

    Contrôler le comportement d'un événement

    Certains éléments ont un comportement par défaut : un lien ouvre une page, un formulaire recharge la page, etc. On peut contrôler cela avec preventDefault et stopPropagation.

    • Empêcher le comportement par défaut : 
      const lien = document.querySelector("a");

lien.addEventListener("click", (e) => {
    e.preventDefault();
    console.log("Lien bloqué");
});
    • Stopper la propagation : 
      const zone = document.querySelector("#zone");
const btn = document.querySelector("#btn");

zone.addEventListener("click", () => {
    console.log("zone");
});

btn.addEventListener("click", (e) => {
    e.stopPropagation();
    console.log("bouton seul");
});

    À utiliser avec parcimonie : stopPropagation peut rendre le code difficile à comprendre si plusieurs écouteurs sont impliqués.

    Erreurs fréquentes

    • Exécuter la fonction au lieu de la passer : l’événement ne fonctionne pas.
    • Oublier que e.target peut être un enfant : mauvais élément manipulé.
    • Mettre un écouteur sur un élément null : erreur si la sélection DOM est mauvaise.
    • Utiliser trop de stopPropagation : logique difficile à suivre.

btn.addEventListener("click", () => {
    log("OK");
});

    3. Écrire du code : coder le programme suivant.

      • Créer une variable compteur initialisée à 0.
      • Ajouter un écouteur click sur un bouton.
      • À chaque clic :
        • incrémenter le compteur,
        • afficher sa valeur dans la console,
        • mettre à jour le texte d’un élément de la page pour afficher le compteur.
      • Si le compteur dépasse 10, ajouter une classe warning.
