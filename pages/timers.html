    Exécuter du code plus tard

    Pour exécuter du code après un délai, on utilise setTimeout. Le navigateur attend la durée demandée, puis exécute la fonction. Le délai est exprimé en millisecondes. setTimeout ne bloque pas le programme : il programme juste une exécution future.

    • Exemple simple : 
      setTimeout(() => {
    console.log("3 secondes plus tard");
}, 3000);
    • Passer une fonction nommée : 
      function direBonjour(prenom) {
    console.log(`Bonjour, $"{prenom}
    "`);
}

setTimeout(() => direBonjour("Alice"), 1000);

    Répéter une action

    Pour exécuter une action régulièrement, on utilise setInterval. Le navigateur exécute la fonction toutes les n millisecondes. setInterval ne bloque pas le code. Il déclenche simplement des exécutions répétées.

    • Exemple : afficher un compteur toutes les secondes : 
      let compteur = 0;

setInterval(() => {
    compteur++;
    console.log(compteur);
}, 1000);
    • Stocker l’identifiant du timer : 
      const timerId = setInterval(() => {
    console.log("tick");
}, 1000);
    • Il est parfois nécessaire de contrôler la répétition. On utilise setTimeout dans une boucle ou une structure conditionnelle. 
      let n = 0;

function tick() {
    n++;
    console.log(n);

    if (n < 5) {
        setTimeout(tick, 1000);
    }
}

setTimeout(tick, 1000);

    Arrêter un timer

    setTimeout et setInterval renvoient un identifiant qui permet d’arrêter le timer. On stocke cet identifiant dans une variable.

    • Arrêter un setTimeout : 
      const id = setTimeout(() => {
    console.log("Ne s'affichera pas");
}, 3000);

clearTimeout(id);
    • Arrêter un setInterval : 
      let compteur = 0;

const id = setInterval(() => {
    compteur++;
    console.log(compteur);

    if (compteur === 5) {
        clearInterval(id);
    }
}, 1000);

    Erreurs fréquentes

    • Penser que setTimeout "bloque" le programme : le code continue de s’exécuter.
    • Exécuter la fonction au lieu de la passer : le timer ne fait pas ce qui est attendu.
    • Oublier de stocker l’identifiant : impossible d’arrêter le timer.
    • Utiliser setInterval alors qu’on veut une répétition contrôlée : risque d’effets en cascade.

    Exercices

    1. Lire du code :

      console.log("A");

setTimeout(() => {
    console.log("B");
}, 1000);

console.log("C");

    2. Lire et écrire du code : corriger les erreurs du code suivant.

      Le compteur doit fonctionner 10 fois.

      let compteur = 0;

setInterval(() => {
    compteur++;
    console.log(compteur);
}, 1000);

    3. Écrire du code : coder le programme suivant.

      • Créer un compteur initialisé à 10.
      • Chaque seconde, décrémenter le compteur et afficher sa valeur dans la console.
      • Quand le compteur atteint 0 :
        • afficher "Terminé",
        • arrêter le timer.
