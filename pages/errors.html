    Intercepter une erreur

    Une erreur peut interrompre un programme. Pour l’intercepter, on utilise try / catch. Le code dans try est exécuté normalement ; si une erreur survient, on passe dans catch. 

    try {
    const x = JSON.parse("{ mauvais json }");
    console.log(x);
} catch (err) {
    console.log("Erreur interceptée :", err.message);
}

    On peut aussi utiliser finally : il s’exécute dans tous les cas (succès ou échec). 

    try {
    console.log("Début");
    throw new Error("Problème");
} catch (err) {
    console.log("Catch :", err.message);
} finally {
    console.log("Enfin : je m'exécute toujours");
}

    try / catch sert à gérer l’imprévu (données invalides, réseau, erreurs de logique).

    Gérer les erreurs réseau

    Avec fetch, il y a 2 types d’erreurs courantes :

    • erreur réseau (pas de connexion, DNS, CORS…) : la Promise passe dans catch.
    • erreur HTTP (404, 500…) : fetch réussit, mais response.ok vaut false.
    fetch("/api/eleves/")
    .then((response) => {
        if (!response.ok) {
            throw new Error("HTTP " + response.status);
        }
        return response.json();
    })
    .then((data) => {
        console.log(data);
    })
    .catch((err) => {
        console.log("Erreur réseau ou HTTP :", err.message);
    });

    La version async / await est souvent plus lisible : 

    async function chargerEleves() {
    try {
        const response = await fetch("/api/eleves/");

        if (!response.ok) {
            throw new Error("HTTP " + response.status);
        }

        return await response.json();
    } catch (err) {
        console.log("Erreur :", err.message);
        return [];
    }
}

    Ici, on renvoie [] en cas d’échec : c’est une stratégie simple et robuste pour éviter de casser la suite du programme.

    Afficher les erreurs à l'utilisateur

    Dans une application, on doit souvent afficher un message clair à l’utilisateur, tout en gardant le détail technique dans la console. On utilise souvent une fonction qui centralise l’affichage. 

    function afficherErreur(messageUtilisateur, err) {
    console.log("Détail :", err);

    const zone = document.querySelector("#error");
    if (zone) {
        zone.textContent = messageUtilisateur;
        zone.classList.add("visible");
    }
}

    Exemple d’utilisation avec fetch : 

    fetch("/api/eleves/")
    .then((response) => {
        if (!response.ok) {
            throw new Error("HTTP " + response.status);
        }
        return response.json();
    })
    .catch((err) => {
        afficherErreur("Impossible de charger les données.", err);
    });

    Créer ses propres erreurs

    On peut créer une erreur volontairement avec throw. C’est utile pour valider des données et arrêter un traitement si une condition n’est pas respectée. 

    function diviser(a, b) {
    if (b === 0) {
        throw new Error("Division par zéro");
    }
    return a / b;
}

try {
    console.log(diviser(10, 0));
} catch (err) {
    console.log(err.message);
}

    Erreurs fréquentes

    • Utiliser try / catch pour masquer un bug :le problème revient plus tard.
    • Oublier de tester response.ok avec fetch : les erreurs 404/500 ne sont pas interceptées comme prévu.
    • Oublier que JSON.parse peut lever une exception : plantage si le JSON est invalide.
    • Attraper une erreur et ne rien faire : débogage difficile.

    Exercices

    1. Lire du code :

      try {
    console.log("A");
    JSON.parse("{bad json}");
    console.log("B");
} catch (err) {
    console.log("C");
} finally {
    console.log("D");
}

    2. Lire et écrire du code : corriger les erreurs du code suivant.

      fetch("/api/eleves/")
    .then((r) => r.json())
    .then((data) => console.log(data))
    .catch((err) => console.log(err));

      L'objectif est de gérer aussi les erreurs HTTP (404/500).

    3. Écrire du code : coder le programme suivant.

      • Écrire une fonction safeParse qui prend une chaîne JSON.
      • Si le JSON est valide, la fonction renvoie l’objet.
      • Si le JSON est invalide, la fonction renvoie un message d’erreur lisible, sous forme d’objet : { data, error }).
