    Créer un objet

    Un objet permet de regrouper plusieurs informations qui décrivent une "même chose". Chaque information est stockée dans une propriété sous la forme clé : valeur.

    • Créer un objet "littéral" : 
      const eleve = {
    nom: "Alice",
    age: 12,
    classe: "6eA"
};
    • Créer un objet vide puis ajouter des propriétés : 
      const eleve2 = {};
eleve2.nom = "Basile";
eleve2.age = 13;

    Un objet est utile pour stocker un groupe de données cohérentes ensemble, et qui doivent être nommées (nom, âge, etc.).

    Accéder aux propriétés

    Il existe deux manières principales d’accéder aux propriétés d’un objet.

    • Notation "point" : 
      const eleve = {
    nom: "Alice",
    age: 12
};

console.log(eleve.nom);  // "Alice"
console.log(eleve.age);  // 12
    • Notation "crochets" (utile si la clé est dans une variable) : 
      const cle = "nom";

console.log(eleve[cle]);  // "Alice"
    • Modifier une propriété : 
      eleve.age = 13;

    Si une propriété n’existe pas, JavaScript renvoie undefined.

    Parcourir un objet

    Pour parcourir un objet, on utilise for...in. Cette boucle renvoie les clés de l’objet.

    • Afficher les clés : 
      const eleve = {
    nom: "Alice",
    age: 12,
    classe: "6eA"
};

for (const cle in eleve) {
    console.log(cle);
}
    • Afficher les clés et les valeurs : 
      for (const cle in eleve) {
    console.log(cle + " : " + eleve[cle]);
}

    Attention : dans ce cas, la notation "point" ne fonctionne pas avec une clé dans une variable.

    Ajouter des méthodes

    Une méthode est une fonction stockée dans une propriété. Elle permet d’associer des comportements à un objet.

    • Définir une méthode : 
      const eleve = {
    nom: "Alice",
    age: 12,
    direBonjour: function () {
        console.log("Bonjour !");
    }
};
    • Accéder à une méthode : 
      eleve.direBonjour();
    • Notation plus courte : 
      const eleve2 = {
    nom: "Basile",
    direBonjour() {
        console.log("Bonjour !");
    }
};
    • Une méthode peut aussi renvoyer une valeur avec return. 
      const calculerAnneeNaissance = (age) => 2026 - age;

    Comprendre this

    this est un mot-clé qui désigne "l’objet courant". Dans une méthode, il permet d’accéder aux propriétés du même objet. 

    const eleve = {
    nom: "Alice",
    age: 12,
    presentation() {
        console.log(`Je m'appelle ${this.nom} et j'ai ${this.age} ans.`);
    }
};

eleve.presentation();

    Comprendre la notion de référence

    En JavaScript, les objets sont manipulés par référence. Cela signifie que la variable ne contient pas l’objet, mais un lien vers lui.

    • Exemple : deux variables pointent vers le même objet : 
      const a = { x: 1 };
const b = a;

b.x = 99;

console.log(a.x);  // 99
    • Comparer deux objets : 
      const o1 = { x: 1 };
const o2 = { x: 1 };

console.log(o1 === o2);  // false

    === compare les références, pas le contenu. Deux objets identiques en apparence ne sont pas égaux s’ils ne sont pas la même référence.

    Créer des objets avec un constructeur

    Pour créer plusieurs objets du même "type", on utilise un constructeur avec new. 

    function Eleve(nom, age) {
    this.nom = nom;
    this.age = age;
    this.presentation = function () {
        console.log(`Je m'appelle ${this.nom}`);
    };
}

const e1 = new Eleve("Alice", 12);
const e2 = new Eleve("Basile", 13);

e1.presentation();
e2.presentation();

    Ici, this représente l’objet créé par new.

    Erreurs fréquentes

    • Confondre notation point et crochets : lecture incorrecte d’une propriété.
    • Utiliser objet.cle au lieu de objet[cle] :renvoie souvent undefined.
    • Oublier que les objets sont des références : modifier un objet "copié" modifie l’original.
    • Utiliser this hors d’une méthode appelée comme objet.methode() :this ne pointe pas vers l’objet attendu.
    • Oublier new avec un constructeur :objet non créé comme prévu.
