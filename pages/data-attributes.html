    Récupérer des données HTML

    Les data-attributes permettent de stocker des informations directement dans le HTML, sur un élément.Ces données ne sont pas visibles par l’utilisateur, mais peuvent être lues en JavaScript. Ils sont écrits sous la forme data-nom dans le HTML. 

    <button data-id="12" data-type="add">
    Ajouter
</button>
    • Le HTML décrit l’interface.
    • Les data-* transportent l’information.
    • JavaScript lit ces données pour décider quoi faire.

    C’est un moyen propre de relier interface et logique, sans mélanger JavaScript directement dans le HTML.

    Lier des données avec dataset

    Les data-attributes permettent de stocker des informations dans le HTML, directement sur un élément, sans les afficher. En JavaScript, on y accède via dataset. Règle de nommage : data-user-id devient dataset.userId (kebab-case → camelCase). 

    <button id="btn" data-user-id="42" data-role="admin">
    Voir profil
</button>
    const btn = document.querySelector("#btn");

console.log(btn.dataset.userId);  // "42"
console.log(btn.dataset.role);  // "admin"

    Relier HTML et JavaScript

    Le modèle le plus courant : on récupère un groupe d’éléments (par exemple une liste de boutons), et on utilise leur data-attributes pour savoir quoi faire quand l’utilisateur clique. 

    <ul>
    <li><button class="btn-produit" data-name="Stylo" data-price="1.5">Ajouter</button></li>
    <li><button class="btn-produit" data-name="Cahier" data-price="2.9">Ajouter</button></li>
</ul>
    const boutons = document.querySelectorAll(".btn-produit");

for (const btn of boutons) {
    btn.addEventListener("click", () => {
        const nom = btn.dataset.name;
        const prix = Number(btn.dataset.price);

        console.log(`Ajout : ${nom} (${prix} €)`);
    });
}

    Ce pattern est très utilisé dans les interfaces : HTML est "déclaratif", JS est "logique".

    Délégation d'événements

    On utilise cette variante quand le nombre d'éléments est plus important. On fait appel aux événements avant de cibler l'objet avec son dataset. 

    <ul>
    <li><button class="btn-produit" data-name="Stylo" data-price="1.5">Ajouter</button></li>
    <li><button class="btn-produit" data-name="Cahier" data-price="2.9">Ajouter</button></li>
</ul>
    const liste = document.querySelector("ul");

liste.addEventListener("click", (element) => {
    if (!element.target.matches(".btn-produit")) {
        return;
    }

    const btn = element.target;
    const nom = btn.dataset.name;
    console.log(`Ajout : ${nom} (${prix} €)`);
});

    Erreurs fréquentes

    • Oublier que dataset renvoie des chaînes : concaténation au lieu de calcul.
    • Confondre data-user-id et dataset.userId : mauvais nom de propriété.
    • Mettre des données "complexes" directement dans un attribut : HTML illisible.
