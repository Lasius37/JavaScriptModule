    Créer et utiliser une fonction

    Une fonction est un bloc de code qu’on peut réutiliser. Elle peut recevoir des données (paramètres) et renvoyer un résultat avec return.

    • Créer une fonction : 
      function direBonjour() {
    console.log("Bonjour !");
}
    • Appeler une fonction (la déclencher) : 
      direBonjour();  // affiche "Bonjour !"
    • Renvoyer une valeur : 
      function carre(x) {
    return x ** 2;
}

let resultat = carre(5);
console.log(resultat);  // 25

    return arrête la fonction immédiatement et renvoie la valeur. Sans return, la fonction renvoie undefined.

    Écrire une fonction fléchée

    Une fonction fléchée (arrow function) est une écriture plus courte. Elle est très utilisée en JavaScript moderne.

    • Exemple simple : 
      const direBonjour = () => {
    console.log("Bonjour !");
};

direBonjour();
    • Un paramètre : 
      const carre = (x) => {
    return x ** 2;
};
    • Version courte (return implicite) : 
      const carre = (x) => x ** 2;

    Si la fonction ne contient qu’une seule expression, on peut supprimer les {} et return.

    Passer des paramètres à une fonction

    Les paramètres permettent de rendre une fonction générique. On passes des arguments au moment de l’appel.

    • Plusieurs paramètres : 
      function somme(a, b) {
    return a + b;
}

console.log(somme(3, 7));  // 10
    • Paramètre optionnel (valeur par défaut) : 
      function saluer(nom = "inconnu") {
    console.log(`Bonjour ${nom} !`);
}

saluer("Paul");
saluer();  // utilise "inconnu"
    • Retourner un résultat “utilisable” : 
      const ttc = (prixHT, taux = 0.2) => prixHT * (1 + taux);

let prix = ttc(100);
console.log(prix);

    Utiliser une fonction comme paramètre

    En JavaScript, une fonction est une valeur : on peut la stocker dans une variable, et la passer comme paramètre. On appelle ça une callback.

    Une callback permet de déléguer un comportement : on transmet à une fonction ce qu’elle devra exécuter à un moment précis. Elles sont très utilisées pour déclencher une action après un événement, une validation ou une opération asynchrone. 

    function executer(action) {
    action();  // on exécute la fonction reçue
}

const direBonjour = () => {
    console.log("Bonjour !");
};

executer(direBonjour);

    Comprendre la portée des variables

    La portée (scope) d’une variable indique où elle est accessible.

    • let et const ont une portée de bloc (entre {}).
    • Une variable déclarée dans une fonction n’existe pas en dehors. 
    function exemple() {
    let x = 10;  // x est locale à la fonction

    if (x > 5) {
        const msg = "x est grand";  // msg existe seulement dans le if
        console.log(msg);
    }

    // console.log(msg); -> ReferenceError (msg n'existe pas ici)
}

exemple();

    Erreurs fréquentes

    • Oublier d’appeler la fonction (()) : la fonction ne s’exécute pas.
    • Confondre return et console.log : on affiche, mais on ne renvoie rien.
    • Utiliser une variable créée dans un bloc en dehors du bloc : ReferenceError.
    • Modifier un paramètre sans vérifier son type : comportement inattendu.
